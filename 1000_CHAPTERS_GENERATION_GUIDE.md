# 1000章小说生成指南

## 重要提示

⚠️ **单次生成限制**：由于API超时限制（5分钟），单次最多只能生成50章。要生成1000章，需要分批进行。

## 分批生成步骤

### 第1步：准备工作

1. **打开应用**：进入网络小说写作助手
2. **生成大纲**：使用"自动生成大纲"功能，设置目标章节数为1000
3. **检查大纲**：确保大纲包含所有必要信息（世界观、人物、剧情等）

### 第2步：分批生成

**第1批（1-50章）**：
1. 打开"批量生成章节"功能
2. 设置生成章节数为50
3. 点击"批量生成章节"按钮
4. 等待30-40分钟，生成完成
5. 等待2-3分钟，确保数据完全保存

**第2批（51-100章）**：
1. 再次点击"批量生成章节"
2. 设置生成章节数为50
3. 点击"批量生成章节"按钮
4. 等待30-40分钟，生成完成
5. 等待2-3分钟，确保数据完全保存

**重复20次，直到达到1000章**

### 第3步：验证数据

1. 每次生成完成后，检查章节列表
2. 确认新章节已正确显示
3. 查看小说总览，确认章节数和总字数
4. 如有异常，刷新页面重新加载

## 时间预估

| 项目 | 时间 |
|------|------|
| 单次生成50章 | 30-40分钟 |
| 数据保存等待 | 2-3分钟 |
| 1000章总耗时 | 12-15小时 |

**建议**：
- 可以分几天完成，每天生成2-3批（约100-150章）
- 夜间生成（晚上睡觉时运行）
- 避免高峰期网络拥堵

## 常见问题

### Q1: 为什么单次只能生成50章？

A: API有5分钟的超时限制，50章需要30-40分钟，已经是极限。如果一次生成更多章节，会导致超时失败。

### Q2: 如果生成中断了怎么办？

A:
1. 检查已生成章节数
2. 从断点继续生成
3. 使用章节列表查看哪些章节已完成
4. 根据当前章节数，继续生成剩余章节

### Q3: 如何确保数据不丢失？

A:
1. 每次生成后等待2-3分钟，让IndexedDB完全保存
2. 定期刷新页面，检查数据是否保存
3. 使用"数据管理中心"查看快照
4. 不要同时打开多个标签页

### Q4: 生成过程中可以做其他事情吗？

A: 可以，但不建议关闭标签页。可以：
- 切换到其他标签页
- 最小化浏览器窗口
- 使用其他软件

### Q5: 如果生成失败怎么办？

A:
1. 检查网络连接
2. 查看错误提示
3. 刷新页面重试
4. 检查API Key配置

### Q6: 可以生成更多或更少的章节吗？

A: 可以：
- 最少1章，最多50章
- 建议每次20-50章
- 可以根据实际情况调整
